Detailed Description

Etch-A-Sketch game implementation.

CSketch inherits from CBase4618 and defines the Etch-A-Sketch behavior. It reads joystick/button inputs (and sensor data where applicable) via the TM4C123G, updates cursor state, and draws to an OpenCV canvas.

Update sketch state based on the latest inputs.

Uses joystick percentages to update cursor motion, updates the line endpoints (pt1, pt2), and manages colour selection / reset logic.

Parameters
[in]percentage_xJoystick X position as a percentage.
[in]percentage_yJoystick Y position as a percentage.
[in]colour_button_pressedTrue if the colour-change button is pressed.
[in,out]pt1Previous/anchor point for drawing.
[in,out]pt2Current point for drawing.
[in,out]indexColour index (selected colour).
[in,out]findexAccelerometer reading (float) forwarded to draw().
Note
index is specifically used as the colour selector for drawing.
findex is specifically used to carry an accelerometer-derived value from update into draw.
