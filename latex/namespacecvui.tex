void cvui::\+begin\+Column (\begin{DoxyParamCaption}\item[{cv::\+\+Mat \&}]{the\+Where}{, }\item[{int}]{theX}{, }\item[{int}]{theY}{, }\item[{int}]{the\+Width}{ = {\ttfamily -\/1}, }\item[{int}]{the\+Height}{ = {\ttfamily -\/1}, }\item[{int}]{the\+Padding}{ = {\ttfamily 0}}\end{DoxyParamCaption})}

Start a new column.

One of the most annoying tasks when building UI is to calculate where each component should be placed on the screen. cvui has a set of methods that abstract the process of positioning components, so you don\textquotesingle{}t have to think about assigning a X and Y coordinate. Instead you just add components and cvui will place them as you go.

You use {\ttfamily \doxylink{namespacecvui_a8f5f6451ab5af26f703760ca2cea9801}{begin\+Column()}} to start a group of elements. After {\ttfamily \doxylink{namespacecvui_a8f5f6451ab5af26f703760ca2cea9801}{begin\+Column()}} has been called, all subsequent component calls don\textquotesingle{}t have to specify the frame where the component should be rendered nor its position. The position of the component will be automatically calculated by cvui based on the components within the group. All components are placed below each other, from the top of the screen towards the bottom.

E.\+g.

Rows and columns can be nested, so you can create columns/\+rows within columns/\+rows as much as you want. It\textquotesingle{}s important to notice that any component within {\ttfamily \doxylink{namespacecvui_a8f5f6451ab5af26f703760ca2cea9801}{begin\+Column()}} and {\ttfamily \doxylink{namespacecvui_aa2115cb75dfb6a2c7dbbb27764741382}{end\+Column()}} {\itshape do not} specify the position where the component is rendered, which is also true for {\ttfamily \doxylink{namespacecvui_a8f5f6451ab5af26f703760ca2cea9801}{begin\+Column()}}. As a consequence, {\bfseries{be sure you are calling {\ttfamily begin\+Column(width, height)} when the call is nested instead of {\ttfamily begin\+Column(x, y, width, height)}}}, otherwise cvui will throw an error.

Start a new row.

One of the most annoying tasks when building UI is to calculate where each component should be placed on the screen. cvui has a set of methods that abstract the process of positioning components, so you don\textquotesingle{}t have to think about assigning a X and Y coordinate. Instead you just add components and cvui will place them as you go.

You use {\ttfamily \doxylink{namespacecvui_aaab319849ff974b6cc67df45ab64ea15}{begin\+Row()}} to start a group of elements. After {\ttfamily \doxylink{namespacecvui_aaab319849ff974b6cc67df45ab64ea15}{begin\+Row()}} has been called, all subsequent component calls don\textquotesingle{}t have to specify the frame where the component should be rendered nor its position. The position of the component will be automatically calculated by cvui based on the components within the group. All components are placed side by side, from left to right.

Rows and columns can be nested, so you can create columns/\+rows within columns/\+rows as much as you want. It\textquotesingle{}s important to notice that any component within {\ttfamily \doxylink{namespacecvui_aaab319849ff974b6cc67df45ab64ea15}{begin\+Row()}} and {\ttfamily \doxylink{namespacecvui_a2e64d25523acde477058cc7f76a6c446}{end\+Row()}} {\itshape do not} specify the position where the component is rendered, which is also true for {\ttfamily \doxylink{namespacecvui_aaab319849ff974b6cc67df45ab64ea15}{begin\+Row()}}. As a consequence, {\bfseries{be sure you are calling {\ttfamily begin\+Row(width, height)} when the call is nested instead of {\ttfamily begin\+Row(x, y, width, height)}}}, otherwise cvui will throw an error.

It is also possible to tell cvui to handle Open\+CV\textquotesingle{}s event queue automatically (by informing a value greater than zero in the {\ttfamily the\+Delay\+Wait\+Key} parameter of the function). The value passed to {\ttfamily the\+Delay\+Wait\+Key} will be used as the delay for {\ttfamily cv::\+wait\+Key()}.
